# Blog — GitHub Pages

Each blog is a **folder** with a `content.md` file. The site is built with Jekyll; GitHub builds and publishes when you push.

## Repo structure

- **One folder per blog**, e.g. `rust-ai-workloads-backend/` with `content.md` inside.
- Run the sync script before committing so `_posts/` is updated and Jekyll can build.

## Adding a new post

1. Create a folder (e.g. `my-new-post/`).
2. Add `content.md` with YAML front matter and Markdown:

```yaml
---
layout: post
title: "Your post title"
date: 2025-03-10 14:00:00 +0000
---

Your content in **Markdown** here...
```

3. Sync into `_posts/` and commit:

```bash
./script/sync_posts.sh
git add _posts/ your-new-folder/
git commit -m "Add post: Your post title"
git push
```

## Sync script

From the repo root:

```bash
./script/sync_posts.sh
```

This reads every top-level folder that has `content.md`, uses the `date` from front matter, and writes `_posts/YYYY-MM-DD-folder-name.md`. Commit the updated `_posts/` after running it.

## Quick start (first-time setup)

1. **Create the repo on GitHub** (e.g. `blog` or `blogs`).
2. **Set `baseurl` in `_config.yml`**:
   - Repo `kkvelan.github.io` → `baseurl: ""` (site at **https://kkvelan.github.io**)
   - Repo `blogs` → `baseurl: "/blogs"` (site at **https://kkvelan.github.io/blogs**)
3. **Push** this folder to the repo.
4. **Enable GitHub Pages**: Repo → **Settings** → **Pages** → Source: **Deploy from a branch** → Branch: **main** → Save.

## Local preview (optional)

```bash
bundle install
bundle exec jekyll serve
```

Open http://localhost:4000/blogs (or http://localhost:4000 if `baseurl` is empty).

## What’s in the repo

| Path | Purpose |
|------|--------|
| `*/content.md` | One folder per blog; source of truth for each post |
| `script/sync_posts.sh` | Copies folder `content.md` → `_posts/` for Jekyll |
| `_posts/` | Generated by sync script; used by Jekyll to build the site |
| `_config.yml`, `_layouts/`, `index.md`, `assets/` | Jekyll site config and layout |

Always edit posts in the **folder** (`content.md`), then run `./script/sync_posts.sh` before committing.

## Viewing style

The site uses a **black background** and **white text** by default. You can add:

- **Images** — `![Description]({{ site.baseurl }}/your-post-folder/image.jpg)`
- **Code snippets / config examples** — use fenced code blocks with an optional language for syntax highlighting:

````markdown
```rust
fn main() {
    println!("Hello");
}
```

```yaml
# config example
server:
  port: 8080
```
````

Rust, YAML, JSON, bash, etc. are supported; code blocks are styled with a dark background and light text.
